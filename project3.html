MULTICLOUD

Shows interconnectivity between AWS and Google cloud. Load and store data in AWS S3 with front end view in Goodle cloud.
AWS | Google Cloud | Terraform


  • View in GitHub

Problem Statement

Luxxy hotel company made a web application for their customers coming to hotel to upload their covid test result to avoid manual checking. Company is finding difficult to store all the customer data and also wants to use this system in other sites as well.

Introduction

Terraform is an infrastructure as code (IaC) tool widely used in cloud computing environments to manage and provision infrastructure resources. It provides a declarative syntax for defining infrastructure components such as virtual machines, networks, storage, and more, allowing for the automated provisioning and management of cloud resources. One of the key advantages of Terraform is its ability to work with multiple cloud providers, including Amazon Web Services (AWS), Microsoft Azure, Google Cloud Platform (GCP), and others. This multi-cloud capability enables organizations to maintain a consistent infrastructure provisioning workflow across different cloud environments, reducing vendor lock-in and increasing flexibility.
Overall, Terraform simplifies and accelerates the process of managing infrastructure in cloud platforms, empowering organizations to efficiently deploy and scale their applications while maintaining control and consistency across their cloud infrastructure.

Solution Architecture

Terraform is used to provision muticloud services between Google Cloud and AWS. We store all the PDF files in AWS S3 bucket and fetch those to app running in google cloud kubernete using IAM user. SQL database will be used to store the customer data (name, home cuuntry, testing type etc.)

Step by step

Create and setup resources

Create terraform user in AWS using IAM service and create access key.


Upload user access key file to Google Cloud for access to AWS.


Enable necessary services in Google cloud like container, API, servicenetworking etc.


Initialize and run Terraform to provision multicloud infrastructure in AWS and Google Cloud.


S3 bucket will be created using terraform.


Create a programmatic user using IAM service in AWS with full access to s3. Create a new user in Google Cloud SQL Instance on Cloud SQL MySQl database with password. Setup database in Google Cloud and host the database using public ip.


Enable Cloud Build API via Cloud Shell. Connect to the GKE (Google Kubernetes Engine) cluster via Console and deploy luxxy app in kubernetes. Now the appliaction is up and running. We can insert customer data using web application but not view or upload PDF in app currently.


In AWS, upload all local PDF files in S3 then sync PDF Files with AWS S3 used for COVID-19 Testing Status System.


Output


Now we can see all the customer files from app view.
